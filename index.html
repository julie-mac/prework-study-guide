HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Before including the head or body though, you must insert the doctype and the language of your site.
  • The title within the head is what shows up to users on the browser's tab or title bar.
  • The meta (or metadata) elements within the head are not visible to users, as it's the information used by the browser itself.
  • Meta elements include the viewport (sets the default view for the page), charset (to set our character encoding) and http-equiv (sets our HTTP header for content).
  • Within the body, there are header elements (h1-h6) that go from biggest to smallest, with h1 being the bigger one always at the top of the page and h6 being the smallest. It's important to go in order here to provide a user-friendly experience.
  • The p element represents a paragraph, which you will use to present most of your text.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items.
  • The img element contains information about a photo displayed on the website, and contains attributes like the src, alt, height and width.
  • The br element creates a line break to separate content.
  • To create a hyperlink, use the a element (stands for anchor!) and href to point to whatever link you'd like to use.

CSS

  • Your selector is whatever element (header, footer, list, etc.) you want to a property to.
  • Properties include color, text-align, margin, padding, height, width, and many more.
  • After every property, you assign a value. For example: header {color:blue;}
  • You can assign the same properties to multiple selectors at once to help reduce the amount of code needed.
  • The margin indicates how much space you want around an element.
  • The padding indicates how much space you want around the content *inside* the element. It's often confused with the margin.
  • The box-shadow property does what it sounds like... It's described by (x,y) offsets relative to the element, as well as blur and spread radius, and colour.
  • Inside your html code, you must always link your css style sheet to it. Place this piece of code below your last meta element and just before the title element of the webpage.
  • Putting a period before a selector (like all the card elements in these sections! ".card") applies whatever property to all elements identifying with the said selector.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone repository-url: clones remote repository onto local computer
  • git remote -v: verifies connection of the repository
  • git add -A: stages changes made to your code
  • git commit -m "descriptive message": commit your staged content as a new commit snapshot
  • git init: initialize an existing directory as a Git repository
  • git pull origin main: pulls the most updated code from the main branch of the remote repository
  • git push origin branch-name: pushes the committed code from your local computer to the repository on github

JavaScript

  • To link your html and javascript, you must enter a link element into the head element of your html code.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Using console.log() displays whatever you put between the brackets in the console.
  • JavaScript reads code in a top-to-bottom manner, so the order of your code matters (Control Flow). A lot!
  • Conditional statements like "if" and "for" are used to disrupt the Control Flow.
  • An if statement can be used so a piece of code only runs if certain conditions are met.
  • If statements can be followed by "else" to place other conditions for different results.
  • There are three types of values: strings, numbers, and boolean (true/false).
  • An array [] can hold different kinds of data, and the positions of the values within them are zero-indexed and sequentia.
  • An example of an array: var shapes = ["triangle", "square", "pentagon", "circle"]; // The triangle is indexed at the 0 position of the shapes array: shapes[0] = triangle
  • A for loop uses a pattern to perform a task on all the items in an array by allowing a single block of code to execute over and over again until there are no items left.
  • A for loop contains three important statements - The starting point; A condition; and The pattern.
  • An example of the for loop: for(var x = 0; x < array.length; x++) {then on the next line you put your code block!}
  • The .length after the array declares the length of the... you guessed it. The array!
  • x++ is shorthand for x+1.
  • A function performs a set of tasks when called.
  • To create a function, you must declare it, like this: function myFunction() {code block on next line}
  • You must always include parentheses after declaring a function, even when nothing is going inside.
  • In order for a function to run, you must call it: myFunction(); // Pretty simple right?
  • If you want to render a piece of code inactive temporarily, you can comment it out by using /* before and */ after the block of code.
